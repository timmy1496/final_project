# Go App — Kubernetes на AWS (Terraform + Ansible)

Повна IaC інфраструктура для Go-застосунку на Amazon EKS.

[infrastructure.puml](infrastructure.puml)

## Архітектура

```
GitHub Actions
     │
     ├── terraform.yml ──→  fmt → validate → scan → plan → apply* → destroy*
     └── deploy.yml    ──→  build → push ECR → deploy EKS
                                          (* ручний запуск)
AWS Infrastructure (Terraform)
┌────────────────────────────────────────────────────────────┐
│  S3 + DynamoDB  — Terraform remote state + locking         │
│                                                            │
│  VPC (10.0.0.0/16)                                        │
│  ├── Public Subnets  [10.0.1.0/24, 10.0.2.0/24]          │
│  │   └── ALB (AWS Load Balancer Controller)               │
│  └── Private Subnets [10.0.10.0/24, 10.0.11.0/24]        │
│      ├── EKS Node Group (t3.medium × 2)                   │
│      │   ├── go-app Deployment (2 replicas + HPA)         │
│      │   └── monitoring (Prometheus + Grafana + Loki)     │
│      └── RDS MySQL (db.t3.micro)                          │
│                                                            │
│  ECR — Docker image registry                               │
└────────────────────────────────────────────────────────────┘
```

## Перший запуск (Bootstrap)

```bash
# 1. Клонувати
git clone <repo> && cd go-k8s-infra

# 2. Налаштувати змінні
cp terraform/environments/dev/terraform.tfvars.example \
   terraform/environments/dev/terraform.tfvars
# Відредагувати terraform.tfvars

# 3. Створити S3 backend (ПЕРШИЙ РАЗ — без backend блоку)
make bootstrap

# 4. Розкоментувати backend блок в terraform/environments/dev/main.tf
# Потім:
make tf-init   # мігрує state в S3

# 5. Розгорнути всю інфраструктуру
make tf-plan
make tf-apply

# 6. Налаштувати сервери (k8s tools + monitoring)
make ansible-deploy
```

## CI/CD Pipeline

### Terraform Pipeline (`terraform.yml`)

| Job | Тригер | Опис |
|---|---|---|
| **1. fmt** | push/PR | `terraform fmt -check` |
| **2. validate + scan** | push/PR | `terraform validate` + tfsec + Checkov |
| **3. plan** | push/PR | `terraform plan`, коментар в PR |
| **4. apply** | manual (`workflow_dispatch`) | Застосування плану |
| **5. destroy** | manual + `confirm_destroy=DESTROY` | Знищення ресурсів |

**Запустити apply вручну:**
```
GitHub → Actions → Terraform Pipeline → Run workflow
  action: apply
  environment: dev
```

**Запустити destroy вручну:**
```
GitHub → Actions → Terraform Pipeline → Run workflow
  action: destroy
  environment: dev
  confirm_destroy: DESTROY
```

### Deploy Pipeline (`deploy.yml`)

```
push до main → Build Docker image → Push to ECR → Trivy scan → Deploy to EKS
```

## Makefile команди

```bash
# Terraform
make tf-fmt           # форматування
make tf-validate      # валідація
make tf-scan          # tfsec + checkov
make tf-plan ENV=dev  # план
make tf-apply         # застосувати (з підтвердженням)
make tf-destroy       # знищити (з підтвердженням DESTROY)

# Ansible
make ansible-deploy      # повний деплой
make ansible-k8s         # тільки k8s tools
make ansible-monitoring  # тільки моніторинг

# Kubernetes
make k8s-status     # статус подів і сервісів
make k8s-logs       # логи застосунку
make grafana-url    # URL + credentials Grafana

# ECR
make ecr-login        # логін в ECR
make ecr-push TAG=v1  # build + push образу
```

## Моніторинг

```bash

# User:     admin
# Password: <auto-generated>
```

## База даних

Пароль RDS зберігається в AWS SSM Parameter Store:

```bash
# Отримати пароль
aws ssm get-parameter \
  --name "/go-app/rds/password" \
  --with-decryption \
  --query Parameter.Value \
  --output text
```
